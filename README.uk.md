# Тренажер холодних дзвінків у нерухомості (голосовий Telegram-бот)

**[English](README.md) · [Deutsch](README.de.md) · Українська**

Голосовий Telegram-бот, що **грає роль власника нерухомості**, щоб співробітники колл-центру
тренували холодні дзвінки. Менеджер «дзвонить» голосовими повідомленнями; бот відповідає
голосом як неохочий власник — висуває реалістичні заперечення, опирається і (інколи)
погоджується на зустріч. Після розмови формує **звіт-оцінку з коучингом**.

> Зроблено як тестове завдання. Діалог та інтерфейс бота — російською (мова цільового
> колл-центру); документація — англійською, українською та німецькою.

## Можливості

- **13 сценаріїв заперечень** (не хоче працювати з агентством, продає сам, комісія завелика,
  негативний досвід, недовіра, «спершу порадитись з сімʼєю» тощо).
- **4 рівні складності** — від теплого власника, що здається після одного заперечення, до
  холодного й відстороненого, що ледь іде на контакт. Градієнт складності **перевірено
  емпірично** (тримається навіть проти адаптивного LLM-«менеджера», не лише фіксованого скрипта).
- **Голос-у-голос** через голосові Telegram: мовлення → текст → діалог → мовлення.
- **Звіт-оцінка** після кожної розмови: 7 критеріїв (0–100), зважений підсумок (%),
  коментарі по кожному критерію, сильні сторони й конкретні рекомендації *з прикладами фраз*.
- **Збереження розмов** (транскрипти, аудіо, звіти) + **інструмент перегляду STT**, щоб звірити,
  що саме почула система розпізнавання.
- **Тримається ролі** — захищена від джейлбрейку / «ти бот?» / сторонніх задач
  (перевірено 48 атаками).

## Як працює

```
Менеджер (голос у Telegram) ─► STT (Groq Whisper API) ─► LLM-діалог (gpt-4o-mini)
                                                         │  персона = сценарій × рівень
       Голос у Telegram ◄── TTS (Silero → OGG/Opus) ◄────┘
   на /finish: повний транскрипт ─► LLM-суддя ─► звіт-оцінка
```

Флоу: `/start` → обрати сценарій → обрати рівень → бот каже *«Алло?»* → говорити голосовими →
`/finish` → отримати звіт. Повний час відповіді: **~3–7 с**.

## Технологічний стек

| Шар | Вибір | Чому |
|-----|-------|------|
| Бот | `aiogram` (Python 3.12) | асинхронний Telegram-фреймворк |
| STT | **Groq Whisper API** (хмара), локальний `faster-whisper` як фолбек | ~1 с і точніше; локально на слабкому CPU було ~10 с |
| LLM діалогу + оцінки | `openai/gpt-4o-mini` через **OpenRouter** | обрано порівнянням 6 моделей за якістю × ціною × надійністю (~$1 / 1000 розмов) |
| TTS | `Silero` (локально) → `ffmpeg` в OGG/Opus | безкоштовно; для проду змінний на OpenAI TTS / ElevenLabs |
| Сховище | SQLite + локальні аудіофайли | нуль налаштувань для демо |

Кожен зовнішній провайдер (STT / LLM / TTS) — за тонким модулем, тож заміна це зміна одного
файлу. Доведено двома замінами: LLM з локального Ollama на OpenRouter і STT з локального
faster-whisper на Groq.

## Налаштування

**Передумови:** Python 3.12, `ffmpeg`, токен Telegram-бота ([@BotFather](https://t.me/BotFather))
і API-ключ [OpenRouter](https://openrouter.ai/keys). Опційно — API-ключ
[Groq](https://console.groq.com/keys) для швидкого хмарного розпізнавання; без нього STT
працює локально через `faster-whisper`.

```bash
# 1. ffmpeg (macOS)
brew install ffmpeg

# 2. віртуальне середовище + залежності
python3.12 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 3. конфіг
cp .env.example .env
#   потім вписати BOT_TOKEN і OPENROUTER_API_KEY (GROQ_API_KEY опційно)

# 4. запуск
.venv/bin/python -m bot.main
```

Перший запуск завантажує модель Silero (TTS) і — без ключа Groq — також локальну модель
Whisper (далі кешуються).

## Деплой

Бот працює як один процес на long-polling і ставиться на невеликий VPS через `systemd` —
ввімкнений компʼютер розробника не потрібен. Оскільки розпізнавання винесене в хмару (Groq),
вистачає скромної машини; помітно памʼяті потребує лише локальний TTS (Silero).

## Тести

```bash
.venv/bin/python -m pytest tests/
```

17 юніт-тестів на логіку звіту, збереження і побудову персони/промпту (без мережі).

## Структура проєкту

```
bot/        код продукту (telegram, stt, llm, tts, persona, analyzer, storage)
tests/      pytest юніт-тести
research/    дослідницькі скрипти валідації моделей, рівнів, судді, guardrail
tools/       stt_review.py — генерує HTML-перегляд «аудіо ↔ транскрипція»
docs/        постановка задачі + архітектура й рішення
```

## Відомі обмеження

- **Роботичний голос** — Silero це безкоштовна заглушка; для проду рекомендуємо OpenAI TTS
  (заміна модуля). Голос — найбільший кошторисний важіль, деталі в архітектурній доці.
- **Оцінки узгоджені, але не калібровані експертом** — корисні як тренувальний фідбек, не
  сертифікована оцінка.
- **Один процес** — паралельні користувачі шикуються в чергу; для проду — пул воркерів + черга.
- **Записи зберігаються безстроково** (навмисно, для архіву) — для проду потрібні згода й
  політика ретеншну (GDPR).
